# ISO 9001 인증서류 자동화 SaaS 기획서

## 1. 제품 개요

### 제품명

가칭: ISO DocPilot

### 한 줄 설명

ISO 9001 품질경영시스템 인증 준비에 필요한 기본 문서와 기록 양식을 AI로 생성하고, 사업장이 직접 수행해야 하는 실제 운영 활동을 단계별로 안내하는 SaaS 웹서비스.

### 핵심 가치

- 인증 준비자가 어떤 서류가 필요한지 한눈에 이해할 수 있다.
- AI가 만들 수 있는 문서는 사업장 정보를 수집한 뒤 초안을 자동 생성한다.
- AI가 대신할 수 없는 실제 운영 사항은 체크리스트와 실행 가이드로 분리해 안내한다.
- 생성된 문서와 사용자의 진행 상태는 Firebase에 저장한다.
- 전체 과정을 따라가면 기본적인 인증 준비 문서와 실제 수행 항목을 함께 확보할 수 있다.

### 대상 사용자

- ISO 9001 신규 인증을 준비하는 중소기업
- 내부 품질담당자 또는 총무/관리 담당자
- 컨설팅 비용을 줄이고 기본 문서 체계를 먼저 갖추려는 사업장
- 인증 컨설턴트가 고객사별 문서 초안을 빠르게 만들고자 하는 경우

## 2. 제품 목표

### MVP 목표

- ISO 9001 인증 준비 흐름을 대시보드에서 단계별로 제공한다.
- 필수/권장 문서를 "AI 생성 가능 문서"와 "사업장 직접 수행 필요 항목"으로 구분한다.
- 각 문서를 클릭하면 필요한 정보를 단계별 질문으로 수집한다.
- 수집된 정보를 기반으로 Gemini 3.0 Flash 모델에 전달할 프롬프트 데이터를 구성한다.
- 생성된 문서는 미리보기, 수정, 저장, 다운로드가 가능해야 한다.
- Firebase Auth, Firestore, Firebase Storage 연동을 전제로 데이터 구조를 설계한다.

### 비목표

- 인증 합격을 보장하지 않는다.
- 심사기관별 세부 요구사항을 완전히 대체하지 않는다.
- 실제 내부심사, 경영검토, 교육훈련, 설비점검 등 사업장에서 수행해야 하는 활동을 AI가 수행했다고 간주하지 않는다.
- 법률 자문 또는 공인 인증 컨설팅으로 표현하지 않는다.

## 3. ISO 9001 문서 범위

ISO 9001:2015에서는 과거 버전처럼 품질매뉴얼이나 특정 6대 절차서가 모두 필수는 아니다. 다만 인증심사에서는 품질경영시스템이 실제로 계획, 실행, 확인, 개선되고 있다는 객관적 증거가 필요하다. 따라서 서비스는 "표준상 명시 문서", "실무상 권장 문서", "운영 기록"을 나누어 제공한다.

### 3.1 AI로 생성 가능한 문서

AI가 사업장 정보를 바탕으로 초안을 만들 수 있는 문서다. 사용자는 초안을 검토하고 회사 상황에 맞게 수정해야 한다.

| 분류 | 문서명 | 목적 | 사용자 입력 필요 정보 |
| --- | --- | --- | --- |
| 기본 체계 | 품질경영시스템 적용범위 | 인증 적용 조직, 사업장, 제품/서비스 범위 정의 | 회사명, 사업장, 제품/서비스, 제외 범위, 외주 범위 |
| 기본 체계 | 품질방침 | 최고경영자의 품질 방향과 고객만족 의지 명문화 | 경영철학, 고객군, 품질 약속, 개선 방향 |
| 기본 체계 | 품질목표 및 추진계획 | 측정 가능한 품질목표와 달성계획 수립 | 목표 항목, 측정지표, 목표값, 담당자, 주기 |
| 조직 이해 | 조직상황 분석표 | 내부/외부 이슈, 기후변화 이슈 포함 | 시장, 고객, 경쟁, 법규, 내부 역량, 기후 관련 영향 |
| 조직 이해 | 이해관계자 요구사항 분석표 | 고객, 임직원, 공급업체, 규제기관 요구사항 정리 | 이해관계자 목록, 요구사항, 대응방안 |
| 프로세스 | 프로세스 맵 | 회사 업무 흐름과 상호작용 정리 | 주요 업무, 입력/출력, 담당부서, KPI |
| 프로세스 | 업무분장표 | 역할과 책임 정의 | 조직도, 직무, 담당 업무, 승인권자 |
| 리스크 | 리스크와 기회 평가표 | 품질목표 달성에 영향을 주는 리스크 관리 | 리스크 항목, 원인, 영향도, 가능성, 대응책 |
| 문서관리 | 문서관리 절차서 | 문서 작성, 승인, 배포, 개정, 폐기 기준 | 승인 구조, 문서번호 체계, 보관 위치 |
| 기록관리 | 기록관리 절차서 | 기록 식별, 보관, 보호, 폐기 기준 | 기록 종류, 보존기간, 담당자, 저장 위치 |
| 교육 | 교육훈련 절차서 | 역량 파악, 교육계획, 효과성 평가 기준 | 직무별 역량, 교육 방식, 평가 기준 |
| 구매/외주 | 외부공급자 관리 절차서 | 공급업체 선정, 평가, 재평가 기준 | 구매 품목, 외주 범위, 평가 기준 |
| 운영 | 제품/서비스 제공 관리 절차서 | 업무 수행 기준과 품질관리 포인트 정리 | 서비스/제품 제공 단계, 검사 기준, 책임자 |
| 검사 | 검사 및 출하 승인 절차서 | 제품/서비스 인도 전 확인 기준 | 검사 항목, 합격 기준, 승인권자 |
| 부적합 | 부적합 및 시정조치 절차서 | 부적합 발견, 원인분석, 재발방지 기준 | 부적합 유형, 처리 권한, 원인분석 방식 |
| 고객 | 고객불만 처리 절차서 | 고객불만 접수, 처리, 재발방지 기준 | 접수 채널, 처리 기한, 담당자 |
| 내부심사 | 내부심사 절차서 | 내부심사 계획, 수행, 보고 기준 | 심사 주기, 심사원, 대상 프로세스 |
| 경영검토 | 경영검토 절차서 | 경영진 검토 입력/출력 기준 | 회의 주기, 참석자, 검토 항목 |
| 양식 | 각종 기록 양식 | 실제 운영 기록을 남길 수 있는 양식 제공 | 문서별 필드, 회사명, 담당부서 |

### 3.2 사업장에서 직접 해야 하는 사항

AI가 문서 초안을 만들 수는 있지만, 아래 활동은 실제로 사업장에서 수행해야 한다. 서비스는 이 항목을 "실행 필요"로 표시하고, 사용자가 수행 기록을 업로드하거나 체크할 수 있게 한다.

| 활동 | 직접 해야 하는 이유 | 서비스 지원 방식 |
| --- | --- | --- |
| 최고경영자 품질방침 승인 | 경영자의 실제 의지와 책임 확인 필요 | 전자 승인, 승인일 기록, 서명 파일 업로드 |
| 품질목표 실행 | 목표 달성 활동은 실제 업무 수행 필요 | 목표별 진행률, 증빙자료 업로드 |
| 교육훈련 실시 | 구성원의 실제 역량 확보 필요 | 교육계획표, 참석자 명단, 평가 기록 양식 |
| 설비/측정장비 관리 | 장비 상태와 검교정은 현장 확인 필요 | 장비대장, 검교정 기록, 알림 |
| 공급업체 평가 | 실제 거래 이력과 품질 이슈 확인 필요 | 평가표 생성, 평가 결과 입력 |
| 고객 요구사항 검토 | 실제 계약/주문/요구사항 검토 필요 | 검토 기록 양식, 변경사항 기록 |
| 제품/서비스 검사 | 실제 산출물 확인 필요 | 검사 기록 양식, 부적합 연결 |
| 부적합 처리 | 실제 문제, 원인, 조치 확인 필요 | 부적합 보고서, 시정조치 추적 |
| 고객불만 처리 | 실제 고객 접점과 해결 결과 필요 | 고객불만 기록, 재발방지 관리 |
| 내부심사 수행 | 조직 운영 상태 확인 필요 | 심사계획, 체크리스트, 결과 보고서 |
| 경영검토 회의 | 경영진이 실제 성과와 개선사항 검토 필요 | 회의록 양식, 안건 자동 구성 |
| 인증심사 대응 | 심사기관 질의에 실제 증거 제시 필요 | 문서 패키지, 증빙 누락 체크 |

## 4. 전체 사용자 흐름

### 4.1 온보딩

1. 사용자가 회원가입 또는 로그인한다.
2. 회사 기본 정보를 입력한다.
3. 인증 준비 상태를 선택한다.
   - 처음 준비 중
   - 일부 문서 보유
   - 내부심사 전
   - 인증심사 예정
4. 업종과 사업 유형을 선택한다.
   - 제조
   - 서비스
   - IT/SaaS
   - 건설/시공
   - 유통
   - 기타
5. 제품/서비스, 사업장, 직원 수, 주요 고객, 외주 여부를 입력한다.
6. 시스템이 ISO 9001 준비 로드맵을 생성한다.

### 4.2 대시보드

대시보드는 사용자가 "지금 무엇을 해야 하는지" 바로 알 수 있게 구성한다.

주요 영역:

- 인증 준비 진행률
- AI 생성 가능 문서 진행률
- 사업장 직접 수행 항목 진행률
- 최근 생성 문서
- 누락된 필수 정보
- 다음 추천 작업
- 심사 준비 체크리스트

상태값:

- 미시작
- 정보 입력 중
- AI 생성 완료
- 사용자 검토 필요
- 승인 완료
- 실행 필요
- 증빙 업로드 완료

### 4.3 문서 라이브러리

문서 라이브러리는 ISO 9001 준비 문서를 카드 또는 테이블 형태로 보여준다.

필터:

- 전체
- AI 생성 가능
- 직접 수행 필요
- 필수
- 권장
- 완료
- 검토 필요

각 문서 카드 구성:

- 문서명
- ISO 관련 조항
- 필수/권장 여부
- AI 생성 가능 여부
- 현재 상태
- 마지막 수정일
- 담당자
- 주요 액션 버튼
  - 정보 입력
  - AI 생성
  - 미리보기
  - 수정
  - 저장
  - 다운로드

### 4.4 문서 생성 흐름

사용자가 문서를 클릭하면 다음 흐름으로 진행된다.

1. 문서 설명
   - 이 문서가 왜 필요한지 설명
   - 심사에서 어떤 증거로 사용되는지 설명
   - AI가 작성할 수 있는 범위와 사용자가 확인해야 할 부분 안내
2. 정보 수집
   - 단계별 질문 폼 제공
   - 기존 회사 정보 자동 불러오기
   - 모르는 항목은 예시 또는 추천값 제공
3. AI 초안 생성
   - Gemini 3.0 Flash 호출 예정
   - 문서 유형별 시스템 프롬프트와 사용자 입력 결합
4. 미리보기
   - 문서 형식으로 렌더링
   - 누락/주의 항목 표시
5. 사용자 편집
   - 텍스트 수정
   - 표 항목 추가/삭제
   - 담당자, 날짜, 승인자 입력
6. 저장
   - Firestore에 문서 메타데이터 저장
   - Firebase Storage에 PDF/DOCX/HTML 저장 예정
7. 다음 작업 추천
   - 관련 문서 생성
   - 필요한 실행 항목 체크
   - 증빙자료 업로드 안내

### 4.5 직접 수행 항목 흐름

직접 수행 항목은 AI 문서 생성 화면과 다르게 "실행 가이드" 중심으로 구성한다.

1. 해야 할 일 요약
2. 왜 필요한지 설명
3. 수행 방법 단계 안내
4. 필요한 증빙자료 안내
5. 관련 양식 다운로드 또는 생성
6. 완료 체크
7. 증빙자료 업로드
8. 담당자와 완료일 저장

예시: 내부심사 수행

- 내부심사 계획 수립
- 심사원 지정
- 프로세스별 체크리스트 생성
- 실제 심사 수행
- 부적합 또는 개선사항 기록
- 내부심사 결과 보고서 저장
- 시정조치로 연결

## 5. 주요 화면 설계

### 5.1 로그인/회원가입

목적:

- 사용자와 회사 워크스페이스를 연결한다.

구성:

- 이메일 로그인
- Google 로그인 옵션
- 회사 워크스페이스 생성
- 약관 동의

### 5.2 회사 설정

목적:

- AI 문서 생성에 반복 사용될 기본 정보를 저장한다.

입력 항목:

- 회사명
- 대표자명
- 사업자등록번호
- 본사 주소
- 사업장 주소
- 업종
- 주요 제품/서비스
- 직원 수
- 부서 목록
- 주요 고객군
- 주요 외주/협력업체 유형
- 인증 적용 범위
- 인증 제외 범위

### 5.3 인증 준비 대시보드

목적:

- 전체 준비 상태를 한 화면에서 이해하게 한다.

UI 구성:

- 상단 진행률 요약
- 준비 단계 타임라인
- 필수 문서 완료율
- 직접 수행 항목 완료율
- 위험/누락 알림
- 다음 작업 CTA

### 5.4 ISO 9001 로드맵

목적:

- 인증 준비를 순서대로 따라갈 수 있게 한다.

단계:

1. 회사 기본정보 정리
2. 조직상황 및 이해관계자 분석

### 5.5 문서 생성 마법사

목적:

- 문서별로 필요한 정보만 자연스럽게 수집한다.

구성:

- 좌측: 단계 목록
- 중앙: 질문 폼
- 우측: 도움말, 예시, ISO 관련 설명
- 하단: 이전/다음/임시저장

질문 유형:

- 단답형
- 장문형
- 선택형
- 다중 선택
- 표 입력
- 담당자 선택
- 날짜 선택
- 파일 업로드

### 5.6 AI 문서 미리보기/편집

목적:

- AI가 만든 문서를 사용자가 최종 확인하고 수정한다.

구성:

- 문서 제목
- 버전
- 작성자
- 검토자
- 승인자
- 본문 편집 영역
- 표 편집 영역
- 변경 이력
- 저장/승인/다운로드 버튼

주의 표시:

- "이 항목은 회사 실제 상황 확인 필요"
- "심사 전 실제 운영 기록이 필요"
- "담당자 승인이 필요"

### 5.7 실행 체크리스트

### 5.8 문서 보관함

목적:

구성:

### 5.9 인증심사 패키지

## 7. AI 기능 설계

### 7.1 AI 모델

예정 모델:

- Gemini 3.0 Flash

설계 원칙:

- 프론트엔드에서는 모델명을 직접 고정하지 않고 백엔드 설정값으로 관리한다.
- 문서 유형별 프롬프트 템플릿을 분리한다.
- 사용자 입력값, 회사 기본정보, 기존 생성 문서를 함께 참조해 일관성을 유지한다.
- 생성 결과에는 "사용자 확인 필요 항목"을 포함한다.

### 7.2 AI 생성 데이터 흐름

1. 사용자가 문서 생성 마법사에서 정보를 입력한다.
2. 프론트엔드가 입력값을 검증한다.
3. API Route로 문서 생성 요청을 보낸다.
4. 서버에서 프롬프트를 구성한다.
5. Gemini 3.0 Flash에 요청한다.
6. 응답을 구조화된 JSON 또는 Markdown으로 받는다.
7. 프론트엔드에서 문서 미리보기로 렌더링한다.
8. 사용자가 수정 및 승인한다.
9. Firestore와 Firebase Storage에 저장한다.

### 7.3 문서 생성 응답 형식

권장 JSON 구조:

```json
{
  "title": "품질방침",
  "documentType": "quality_policy",
  "isoClauses": ["5.2"],
  "content": {
    "sections": [
      {
        "heading": "품질방침",
        "body": "..."
      }
    ]
  },
  "tables": [],
  "userReviewRequired": [
    "대표자의 최종 승인 여부를 확인하세요.",
    "회사 실제 고객 요구사항과 표현이 맞는지 검토하세요."
  ],
  "relatedActions": [
    "품질방침을 조직 내에 공지",
    "품질목표와 연결 여부 확인"
  ]
}
```

## 11. 문서별 질문 설계 예시

- 회사명은 무엇인가?
- 주요 고객은 누구인가?
- 제공하는 제품 또는 서비스는 무엇인가?

- 지속적 개선을 위해 중점적으로 관리할 영역은 무엇인가?
- 품질방침 승인자는 누구인가?

AI 생성 결과:

- 품질방침 본문
- 임직원 공유 문구
- 품질목표와 연결할 수 있는 핵심 방향
- 대표자 승인 필요 안내

## 12. 사용자가 자연스럽게 따라가는 인증 준비 여정

서비스는 문서 목록만 보여주지 않고, 다음과 같은 흐름을 제공해야 한다.

1. "우리 회사 정보 입력"
2. "인증 범위 정하기"
3. "조직상황과 이해관계자 정리"
4. "품질방침과 목표 만들기"
5. "업무 프로세스와 책임 정리"
6. "절차서와 양식 만들기"
7. "실제로 운영하고 기록 남기기"
8. "내부심사 실행"
9. "경영검토 실행"
10. "인증심사 패키지 준비"

각 단계는 완료 조건을 가져야 한다.

예시:

- 품질방침 단계 완료 조건
  - AI 초안 생성 완료
  - 사용자 검토 완료
  - 대표자 승인일 입력
  - 임직원 공유 여부 체크

## 13. 리스크 및 주의사항

- ISO 인증은 문서만으로 완료되지 않는다. 실제 운영 증거가 필요하다.
- AI 생성 문서는 초안이며, 회사 실제 업무와 맞지 않으면 심사에서 부적합이 될 수 있다.
- 심사기관, 업종, 고객 요구에 따라 추가 문서가 필요할 수 있다.
- 개인정보, 사업기밀, 고객정보 입력 시 보안 정책이 필요하다.
- AI 응답 결과는 사람이 승인한 뒤 공식 문서로 확정해야 한다.
